# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

An AstroDB database of unique Gaia DR3 stellar stream members (80,162 stars across 92 streams),
built using the `astrodbtoolkit` ecosystem. The repo defines a database schema (Felis YAML),
data (JSON), and the test suite that validates them.

## Build artifacts

All outputs from astrodb build skills (parsed table results, schema match reports, generated
schema YAML, workflow documentation, directions) are stored in `astrodb-build-artifacts/`.
Ingestion scripts are stored in `astrodb-ingest-artifacts/`. Always write skill outputs to
these folders — do not scatter them to `tmp/` or the project root.

Key files:
- `astrodb-build-artifacts/directions.md` — dataset decisions and ingestion notes
- `astrodb-build-artifacts/workflow.md` — step-by-step build log with prompts used

## Commands

There is no `pyproject.toml` in this repo — install dependencies directly:

```bash
uv pip install astrodbkit astrodb-utils lsst-felis pytest pytest-cov
```

Run the full test suite:

```bash
uv run pytest -p no:warnings tests
```

> **Note:** The `run_tests.yml` CI workflow requires a `GH_TOKEN` secret set in the repo's
> GitHub Actions secrets. Without it, the workflow will fail on checkout steps that need
> authenticated access.

Run a single test file or test:

```bash
uv run pytest tests/test_contents_sources.py
uv run pytest tests/test_contents_sources.py::test_source_names
```

Regenerate the per-table markdown docs in `docs/schema/` from `schema.yaml`:

```bash
uv run python scripts/build_schema_docs.py
```

## Architecture

**Source of truth is `schema.yaml`**, a Felis-format schema defining every table, column,
datatype, and foreign key. `database.toml` points at this schema and at `data/`, and lists
which tables are lookup tables.

**Data lives as JSON:**

- [data/source/](data/source/) — one JSON file per astronomical object
- [data/reference/](data/reference/) — JSON files for lookup tables

**The build pipeline** (`astrodb_utils.build_db_from_json`) reads `schema.yaml` + `database.toml`,
creates a fresh SQLite database, and loads every JSON file in `data/`. The `db` fixture in
`tests/conftest.py` is session-scoped and autouse.

**Schema docs** in `docs/schema/` are auto-generated by `scripts/build_schema_docs.py` from
`schema.yaml`. Do not commit hand-edited docs there — regenerate them after any schema change.
